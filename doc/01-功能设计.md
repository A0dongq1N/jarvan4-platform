# 压力测试平台 — 功能设计

## 零、项目管理

> 平台支持多项目隔离。用户登录后首先进入项目选择页，所有任务、脚本、报告均归属于某个项目。
> Worker 节点为全局共享资源，不属于任何项目。

| # | 功能 | 说明 |
|---|------|------|
| 0.1 | 项目列表 | 展示当前用户有权限访问的项目列表，支持关键词搜索 |
| 0.2 | 新建项目 | 填写项目名称（必填）、描述（选填），创建后可立即进入 |
| 0.3 | 删除项目 | 删除项目及其下所有任务、脚本、报告（级联删除） |
| 0.4 | 项目选择落地页 | 登录后的第一个落地页（blank layout，无侧边栏），选择项目后进入对应工作区 |
| 0.5 | 当前项目切换 | TopBar 左侧展示当前项目名称，点击可返回项目选择页重新切换 |

## 一、测试任务管理

| # | 功能 | 说明 |
|---|------|------|
| 1.1 | 创建压测任务 | 配置任务基本信息（名称、描述、绑定环境） |

## 二、压测脚本管理

> 脚本是压测的核心执行单元，描述"压什么、怎么请求、如何断言"。
> 脚本使用 **Go 语言**编写，在独立 Git 仓库（scripts-repo）中用本地 IDE 开发，push 后由 CI 自动编译并将产物上传对象存储，平台从对象存储分发给 Worker 执行。
> 脚本与任务解耦，一个脚本可被多个任务复用，版本管理通过 Git commit hash 追踪。
> 后续可扩展支持 JS 等语言。

| # | 功能 | 说明 |
|---|------|------|
| 2.1 | 脚本列表 | 展示 CI 已发布的脚本，分页、搜索、筛选（按名称/提交者） |
| 2.2 | 脚本版本列表 | 查看某脚本的历史 commit 版本列表，展示 commit message、提交者、时间 |
| 2.3 | 删除脚本 | 下线未被任务引用的脚本版本 |
| 2.4 | 脚本调试 | 单次试跑指定版本（不计入压测统计），查看请求/响应/日志 |

---

## 三、任务脚本配置

> 将脚本绑定到任务，配置权重，支持多脚本混合执行。
> 任务绑定层**不锁定版本**，始终使用脚本最新发布版本；执行版本在启动压测时快照，写入执行记录和报告，保证执行过程可追溯。

| # | 功能 | 说明 |
|---|------|------|
| 3.1 | 绑定脚本到任务 | 从平台脚本列表选择脚本，绑定后始终使用该脚本最新发布版本 |
| 3.2 | 配置执行权重 | 多个脚本按权重比例混合执行，模拟真实流量分布 |
| 3.3 | 执行版本快照 | 启动压测时自动记录每个脚本当前的 commit hash，写入执行记录和报告，供事后追溯 |

## 四、压测场景配置

> 场景配置描述"怎么压"，脚本描述"压什么"，两者分离。

| # | 功能 | 说明 |
|---|------|------|
| 4.1 | VU 阶梯模式 | 配置任意阶梯数组，每阶段独立设置目标并发数、爬坡时长（从上阶段线性增至目标并发，0=瞬变）和稳定持续时长 |
| 4.2 | 定速 RPS 模式（固定） | 以恒定速率注入请求，令牌桶限速 |
| 4.3 | 定速 RPS 模式（阶梯爬升） | 配置多阶段 RPS 目标值，每阶段含爬坡时长（从上阶段值线性递增）和稳定时长 |
| 4.4 | 曲线实时预览 | 配置页实时渲染压力曲线（爬坡为斜线，稳定为横线，阶梯为斜升或阶跃） |
| 4.5 | 持续时长配置 | 设置压测总时长（秒）；VU 阶梯模式总时长由各阶段累加得出 |
| 4.6 | 请求超时配置 | 单次请求超时阈值（ms） |
| 4.7 | 熔断保护 | 可选开启，压测过程中滑动窗口内错误率超过阈值时自动停止，支持配置错误率阈值、统计窗口（秒）、最少请求数（避免冷启动误触发），触发后状态标记为「熔断停止」区别于手动停止 |

## 五、压测执行控制

| # | 功能 | 说明 |
|---|------|------|
| 5.1 | 启动压测 | 立即执行或定时执行；从任务详情页跳转时自动触发，无需二次点击 |
| 5.2 | 初始化阶段展示 | 启动后进入 pending 阶段，逐步展示「选定 Worker」「下发脚本」「加载插件」「注入流量」四个初始化步骤的实时进度，附带 Worker 节点和脚本版本详情 |
| 5.3 | 停止压测 | 手动终止正在运行的任务（pending / running 阶段均可停止） |
| 5.4 | 定时调度 | cron 表达式配置定时自动压测 |
| 5.5 | 分布式节点调度 | Master 根据场景配置自动估算所需 Worker 数，按负载评分选取节点；可用节点不足时降级运行（缩减目标并发/RPS）并告警；也支持手动指定 Worker 台数 |

## 六、实时监控看板

| # | 功能 | 说明 |
|---|------|------|
| 6.1 | 实时 QPS 曲线 | 折线图展示每秒请求数 |
| 6.2 | 实时响应时间 | P50 / P90 / P99 / Max 延迟曲线 |
| 6.3 | 成功率/错误率 | 实时成功/失败比例 |
| 6.4 | 并发数/推算并发曲线 | VU 模式展示活跃 goroutine 数；RPS 模式展示由 Little's Law 推算的并发数（C = RPS × RT） |
| 6.5 | 接口维度实时指标 | 压测过程中实时展示各接口的 P50/P75/P90/P95/P99、错误率、请求数；点击接口行展开该接口的 RPS / RT / 错误率趋势折线图（滑动窗口 60 点，与全局同频 2 秒更新） |
| 6.6 | 实时错误分析 | 按错误码聚合展示饼图 + 表格（业务错误 / 系统错误分类，含出错次数和占比） |
| 6.7 | 实时日志流 | 展示脚本内 ctx.Log 输出的日志 |

## 九、用户与权限

### 登录

| # | 功能 | 说明 |
|---|------|------|
| 9.1 | 登录页入口 | 未登录时任何页面均重定向到 `/login`；已登录时访问 `/login` 自动跳转至项目选择页 |
| 9.2 | 双模式登录 | 登录页顶部 Tab 切换两种登录方式，带滑动下划线动画；默认展示账号密码模式 |
| 9.3 | 账号密码登录 | 输入用户名 + 密码，字段均必填；登录成功后跳转项目选择页 `/project` |
| 9.4 | 手机号验证码登录 | 输入 11 位手机号 + 6 位短信验证码；手机号格式不合法时「获取验证码」按钮不可点击；验证码有效期 5 分钟，同一手机号 60 秒内禁止重发（按钮显示倒计时）；验证通过后同样跳转项目选择页 |
| 9.5 | 登出 | 清除本地 Token 和用户信息，跳转至登录页；服务端将对应 Token 加入黑名单 |

## 技术栈说明

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue3 + TypeScript + ECharts |
| 后端 | Go + trpc-go（http_no_protocol 模式） |
| 压测脚本 | Go 语言（通过 Go plugin 机制动态加载执行），预留 JS 扩展 |
| 实时数据 | 前端每 1 秒轮询接口 |
| 持久化 | MySQL |
| 缓存/实时状态 | Redis |
| 分布式压测 | Master-Worker 架构，多节点分布式执行 |
